# 타 센터 보고서 — 재사용 프롬프트

아래 블록을 그 센터의 Claude 세션에 **그대로 붙여넣으면** 삽교/내포와 동일한 카드형 보고서가 나옵니다.
(전제: 그 센터 폴더에 `build_daily_all.py`가 복사돼 있고, 그날 케어포 3-6 PDF가 다운로드돼 있음)

---

## 프롬프트 A — PDF만 주고 보고서까지 (권장)

```
너는 우리 센터의 당일 특이사항 보고서를 만든다. 양식은 build_daily_all.py가 정하며 절대 손대지 않는다.
절차:
1. 다운로드 폴더의 최신 "장기요양급여 제공기록지*.pdf"를 pdfplumber로 읽는다(페이지당 수급자 1명).
2. 각 페이지에서 이름 / "혈압/체온 NNN-NN/36.5" / 특이사항 원문을 뽑는다.
   - 특이사항은 케어포 입력 원문 그대로. 임의로 "특이사항 없음"으로 줄이지 마라.
   - "일정없음" 페이지는 absent로 분류.
3. 아래 형식으로 cfg.json을 만든다:
   {
     "date": "YYYY.MM.DD", "weekday": "요일 한 글자",
     "people": [["이름","혈압 NNN-NN / 체온 36.5","특이사항 원문"], ...],  // PDF 순서 전원
     "absent": ["결석자", ...],
     "gyeolseok": "결석자 한 줄 설명"
   }
4. python3 build_daily_all.py cfg.json > YYYYMMDD_특이사항_보고서.html 실행.
5. 출력 HTML을 원본 PDF와 대조해 이름·혈압·체온·인원수가 일치하는지 점검하고 어긋나면 수정한다.
규칙: 이름·수치·약물·증상은 손대지 않는다. 태그·형광펜·발열강조는 스크립트가 자동으로 붙인다.
```

## 프롬프트 B — 이미 정리한 데이터로 보고서만

```
아래 데이터로 우리 센터 당일 특이사항 보고서를 만들어라.
양식은 build_daily_all.py를 그대로 쓰고, 이 cfg를 넣어 실행만 한다:
python3 build_daily_all.py cfg.json > 보고서.html

cfg.json:
{
  "date": "2026.07.20",
  "weekday": "월",
  "people": [
    ["홍길동", "혈압 130-80 / 체온 36.6", "오전 프로그램 참여 양호."],
    ["김영희", "혈압 145-92 / 체온 37.1", "미열 있어 수분 섭취 권장, 오후 안정."]
  ],
  "absent": ["박철수"],
  "gyeolseok": "박철수 (병원 진료로 미이용)"
}
실행 후 원본과 대조까지 해라.
```

---

## 붙여넣을 때 딱 3가지만 확인
1. `build_daily_all.py`가 그 센터 폴더에 있는가 (없으면 삽교 폴더에서 복사).
2. `people`은 **급여제공 전원, PDF 순서대로** 넣었는가.
3. 혈압·체온 문자열에 **체온 숫자**가 있는가 (발열 붉은 테두리 자동 처리의 조건).

## [설계 노트]
- 프롬프트를 A(PDF→보고서 전자동)·B(데이터→보고서)로 나눈 이유: 센터마다 PDF 추출을 Claude에 맡길지, 사람이 정리한 데이터를 줄지 상황이 다르기 때문. A가 손이 덜 가고 권장.
- 양식 동일성의 열쇠는 "build_daily_all.py를 손대지 않는다"는 한 문장. 프롬프트에 이 제약을 명시해 각 센터에서 임의로 HTML을 새로 그리지 않도록 막았다.
